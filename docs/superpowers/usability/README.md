# UI操作性の計測（代表タスク × 操作コスト）

「使いやすさ」を主観でなく**操作コスト（作業に要するアクション数）**で数値化し、版間で追跡する仕組み。
値が増える＝操作が増える＝操作性の悪化、と機械的に判別する。

## 方法（A＋B）

- **A. 代表タスク**：実業務に近い手順を固定（下記）。これが計測の単位。
- **B. 自動リプレイ計測**：各タスクを Playwright で実行し、実 mousedown 数 / keydown 数 / ツール切替回数 / Undo 回数 / エラートースト数 / 要素増減 / 所要 ms を記録。`expect(budget)` で回帰（操作増）を検出。

実行:

```bash
npm run test:usability      # 代表タスクの操作コストを計測（端末に "USABILITY ..." 行で出力）
```

ハーネス本体: [`e2e/usability.spec.js`](../../../e2e/usability.spec.js)

## 代表タスク（現行）

| # | タスク | ねらい（どの操作性を測るか） | 期待コスト |
|---|--------|------------------------------|-----------|
| 1 | 充電器3台をまとめて移動 | 複数選択＋一括移動の効率 | 矩形選択1＋ドラッグ1＝**2ジェスチャ**（旧来は3回個別移動） |
| 2 | 同種5台のスタンド種別を一括変更 | 同型一括編集の効率 | 矩形選択1＋ドロップダウン1＝**2操作**（旧来は5回個別編集） |
| 3 | 1台をコピーして2箇所へ配置 | コピー＆ペースト＋追従配置の効率 | Ctrl+C・Ctrl+V×2・配置クリック×2 |

## 指標の読み方

- **clicks / keys**：少ないほど良い。同じ成果をより少ない操作で出せているか。
- **toolSwitches**：道具の持ち替え回数。多いと「モード行き来」の負荷が高い兆候。
- **undos / errors**：誤操作・手戻りの量。多いと分かりにくさ・操作ミスの兆候。
- **ms**：体感速度の目安（環境依存なので絶対値より傾向で見る）。

新機能（複数選択・一括編集・コピペ・矢印nudge・選択ズーム）を入れる前後で同じタスクのコストを比べると、
改善が数値で確認できる。バジェットを超えたら回帰として CI/ローカルで気づける。

## 併用すると良い他の手法

- **ヒューリスティック評価（Nielsen 10原則）**：体系的な定性レビュー。エージェントで定期実行。
- **軽量インストルメンテーション（dev限定・ローカルのみ）**：実利用中のツール切替/Undo/エラーを記録して
  自分のボトルネックを可視化（外部送信なし）。
- **実ユーザーの think-aloud 観察**：定性の決定版（人手が要る）。

> このハーネスは「操作コスト＝定量」、ヒューリスティック/観察は「分かりやすさ＝定性」。両輪で判別する。
